# VEDIC_AI_SOURCE_OF_TRUTH.md

## Purpose
Defines the official source assets consumed by Vedic-AI.

---

# Primary Source Assets

## 1. machine_index.json
**Role:** Navigation Brain
Provides Titles, Subtitles, Family Headers, and Page Ranges.
Used For Content discovery, Page lookup, and Semantic routing.

## 2. canonical_content.json
**Role:** Knowledge Brain
Provides Astrology Text, Dasha Data, Shadbala Data, Ashtakavarga Data, Predictions, Remedies, and Tables.
Used For all Engine calculations (Planet Strength, Rasi Strength, Dasha, Transit, Question Engine).

---

# Backend Schemas
The mathematical engines are strictly typed. The `PipelineRunner` guarantees an output dictionary containing:
- `planets`
- `houses`
- `rasis`
- `dashas`
- `vargas`
- `ashtakavarga`
- `yogas`
- `transit`
- `natal_promise`
- `master_probability`
- `metrics`

---

# API Contracts
The `FastAPI` layer guarantees validation via `Pydantic`.
The Frontend `React` layer strictly mirrors these contracts via `src/types/schema.d.ts`.

---

# Runtime Rule
Vedic-AI must NEVER parse PDF files directly.
Vedic-AI operates exclusively on structured JSON outputs generated by HoroscopeCleaner_Final.
All state is held locally (Zustand) or processed statelessly (FastAPI). No database exists in Version 1.
