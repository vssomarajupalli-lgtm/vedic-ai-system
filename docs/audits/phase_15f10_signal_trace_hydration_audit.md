# Phase 15F.10 – Signal Trace Hydration Audit

## 1. Exact `isolated_signals` payload reaching DisplayFormatter
Because of a bug in `queries.py` where `internal_payload` is passed instead of `engine_outputs`, the SignalTranslator fails to pull the primary planet data. This causes `FormulaEvaluator`'s recursive search to rely on the deeply nested JSON tree, where the `ashtakavarga` BAV dictionary (which is processed last) overwrites the original `planets` dictionary.

The exact payload reaching `DisplayFormatter` for Moon and Venus is:
```json
{
  "moon": {
    "house": 2,
    "bindus": 4,
    "modifier": 0
  },
  "venus": {
    "house": 12,
    "bindus": 4,
    "modifier": 0
  }
}
```
*(Notice the complete absence of `"final_score"` or `"breakdown"`).*

## 2. Exact `isolated_signals` payload reaching QuestionResultCard
The `QuestionResultCard` frontend component does not receive the raw `isolated_signals` dictionary. It receives the parsed string arrays generated by `DisplayFormatter`:
```json
"attention_factors": [
  "Moon lacks strength (0/100).",
  "Venus lacks strength (0/100)."
]
```

## 3. Verify whether final_score values are preserved
**No, they are destroyed.** The rich `{"final_score": 71}` dictionaries created by `PlanetStrengthEngine` are preserved in the raw `engine_outputs`, but they are wiped out during `FormulaEvaluator.extract_signals` when the recursive dictionary extraction allows later engines (specifically Ashtakavarga) to overwrite matching keys.

## 4. Identify where 0/100 values are introduced
The literal `0/100` string is introduced in `backend/app/formatters/display_formatter.py` (Line 121):
```python
score = signal_data.get("final_score", 0)
```
Because the `ashtakavarga` dictionary for `"moon"` only contains `{"house", "bindus", "modifier"}`, the `.get()` method safely defaults to `0`. Since `0 < 50`, the formatter classifies it as a weakness and generates `"Moon lacks strength (0/100)."`.

## 5. Confirm whether isolated_signals is being overwritten, truncated, normalized, or translated incorrectly
**It is both translated incorrectly AND overwritten.**

1. **Translated Incorrectly**: In `app/api/v1/endpoints/queries.py` (Line 123), the endpoint incorrectly passes the top-level `internal_payload` to `SignalTranslator`, rather than `engine_outputs`. Because `"planets"` is nested inside `"engine_outputs"`, `SignalTranslator` cannot find it, fails to translate the planets, and fails to append them to the top level of the payload.
2. **Overwritten**: Because the planets are not safely appended to the top level, `FormulaEvaluator` relies entirely on its recursive deep-search. The `safe_set` protection added in Phase 15F.9E prevents primitive (integer) overwrites, but it allows dictionary-on-dictionary overwrites. Because `engine_outputs["ashtakavarga"]["planet_bav_support"]["moon"]` is a dictionary, and because `ashtakavarga` appears later in the JSON tree than `planets`, it permanently overwrites the true planet score dictionary.

## 6. Full Signal Trace Table

| Layer | State of "Moon" | Location / Action |
|-------|-----------------|-------------------|
| **Engine Output (Planets)** | `{"final_score": 71, "breakdown": {...}}` | `internal_payload["engine_outputs"]["planets"]["moon"]` |
| **Engine Output (Ashtakavarga)** | `{"house": 2, "bindus": 4, "modifier": 0}` | `internal_payload["engine_outputs"]["ashtakavarga"]["planet_bav_support"]["moon"]` |
| **SignalTranslator** | *Fails to translate* | Searches `internal_payload["planets"]` (which doesn't exist). |
| **Formula Evaluator** | `{"house": 2, "bindus": 4, "modifier": 0}` | Recursively searches payload. `ashtakavarga` dictates the final dict. |
| **DisplayFormatter** | `score = 0` | `.get("final_score", 0)` |
| **Question Schema** | `attention_factors = ["Moon lacks strength (0/100)."]` | API JSON Response |
| **QuestionResultCard** | Displays: `Moon lacks strength (0/100)` | Frontend React Component |

## 7. Determine whether Signal Trace Console can now be exposed
**No.** The `isolated_signals` payload is fundamentally corrupted before it leaves the backend. If the Signal Trace Console were exposed now, the frontend would attempt to render `final_score` and `breakdown` visualizations using data that actually contains BAV bindus or Varga dignities. 

To fix this, `queries.py` must pass `engine_outputs` to `SignalTranslator`, and `FormulaEvaluator.extract_signals` requires a structural redesign to prevent dictionary collisions.
