# PHASE 9 STEP 2 – DASHA VISIBILITY FIX
**Date:** 2026-06-19

## Root Cause
The `ExecutiveSummarySection.extract()` method inside the `ReportBuilder` architecture was incorrectly referencing a hallucinated path (`dashas.get("current_dasha", {}).get("mahadasha")`), completely ignoring the authentic `DashaEngine` schema output (`synthesis.active_md`, `active_ad`, `active_pd`). As a result, valid Dasha data generated by the backend engine was discarded right at the report formatting layer, defaulting to "Unknown". Furthermore, unit tests were validating against a falsified mock payload matching the wrong path rather than the authentic engine schema.

## Files Modified
1. `backend/app/reports/sections/extractors.py`: Adjusted `ExecutiveSummarySection` to accurately target `engine_outputs["dashas"]["synthesis"]` and retrieve the actual string values for `active_md`, `active_ad`, and `active_pd`.
2. `backend/app/reports/templates/base.html`: Added discrete layout rows for "Current Mahadasha", "Current Antardasha", and "Current Pratyantardasha" inside the Executive Summary card to present these values visibly in the HTML/PDF rendering.
3. `backend/tests/test_report_builder.py`: Replaced the hallucinated mock schema with the legitimate `synthesis` dictionary structure to ensure the test correctly enforces the genuine data contract.

## Validation
- **Tests**: Ran `pytest backend/tests/`. All 613 tests passed gracefully in 0.76s.
- **Report JSON**: Dasha timeline properly bound. 
- **Report HTML/PDF**: Successfully constructs the strings `Current Mahadasha`, `Current Antardasha`, and `Current Pratyantardasha` with the actual mapped names instead of "Unknown".
